# 基于文件的待办事项跟踪技能

## 概述

`todos/` 目录包含一个基于文件的跟踪系统，用于管理代码审查反馈、技术债务、功能请求和工作项。每个待办事项都是一个带有 YAML frontmatter 和结构化部分的 markdown 文件。

在以下情况下应使用此技能：
- 从调查结果或反馈创建新的待办事项
- 管理待办事项生命周期（待处理 → 就绪 → 完成）
- 对待处理项目进行分类以供批准
- 检查或管理依赖关系
- 将 PR 注释或代码发现转换为跟踪的工作
- 在待办事项执行期间更新工作日志

## 文件命名约定

待办事项文件遵循此命名模式：

```
{issue_id}-{status}-{priority}-{description}.md
```

**组件：**
- **issue_id**：顺序号（001、002、003...）- 永不重用
- **status**：`pending`（需要分类）、`ready`（已批准）、`complete`（完成）
- **priority**：`p1`（关键）、`p2`（重要）、`p3`（有则更好）
- **description**：kebab-case，简要描述
