# 在技能中使用模板

<purpose>
模板是可复用的输出结构，Claude 会复制并填充它们。它们确保输出一致、高质量，而无需每次重新生成结构。
</purpose>

<when_to_use>
在以下情况使用模板：
- 输出应在不同调用中保持一致的结构
- 结构比创意生成更重要
- 填充占位符比从空白页生成更可靠
- 用户期望可预测、专业外观的输出

常见模板类型：
- **计划** - 项目计划、实施计划、迁移计划
- **规范** - 技术规范、功能规范、API 规范
- **文档** - 报告、提案、摘要
- **配置** - 配置文件、设置、环境设置
- **脚手架** - 文件结构、样板代码
</when_to_use>

<template_structure>
模板位于技能目录内的 `templates/` 中：

```
skill-name/
├── SKILL.md
├── workflows/
├── references/
└── templates/
    ├── plan-template.md
    ├── spec-template.md
    └── report-template.md
```

模板文件包含：
1. 清晰的章节标记
2. 占位符指示符（使用 `{{placeholder}}` 或 `[PLACEHOLDER]`）
3. 内联指导，说明每个位置应填写什么内容
4. 在有帮助的地方提供示例内容
</template_structure>

<template_example>
```markdown
# {{PROJECT_NAME}} 实施计划

## 概述
{{1-2 句话总结此计划涵盖的内容}}

## 目标
- {{主要目标}}
- {{次要目标...}}

## 范围
**范围内：**
- {{包含的内容}}

**范围外：**
- {{明确排除的内容}}

## 阶段

### 阶段 1：{{阶段名称}}
**持续时间：** {{预估持续时间}}
**交付成果：**
- {{交付成果 1}}
- {{交付成果 2}}

### 阶段 2：{{阶段名称}}
...

## 成功标准
- [ ] {{可衡量标准 1}}
- [ ] {{可衡量标准 2}}

## 风险
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|------------|--------|------------|
| {{风险}} | {{高/中/低}} | {{高/中/低}} | {{策略}} |
```
</template_example>

<workflow_integration>
工作流引用模板的方式如下：

```xml
<process>
## 步骤 3：生成计划

1. 读取 `templates/plan-template.md`
2. 复制模板结构
3. 根据收集的需求填充每个占位符
4. 检查完整性
</process>
```

工作流告诉 Claude **何时**使用模板。模板提供 **什么**结构来生成。
</workflow_integration>

<best_practices>
**应该做：**
- 保持模板专注于结构，而非内容
- 一致地使用清晰的占位符语法
- 在章节可能模糊的地方添加简短的内联指导
- 使模板完整但最小化

**不应该做：**
- 放入可能被逐字复制的过多示例内容
- 为真正需要创意生成的输出创建模板
- 使用太多必需章节过度约束
- 需求变更时忘记更新模板
</best_practices>
